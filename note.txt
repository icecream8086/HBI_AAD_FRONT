TS 代码分析工具与优化方案 — 前端适用性评估
=============================================

项目概况：Vue 3 + TypeScript SPA，约 9500 行源码，27 个页面，
0 个复用组件，后端是独立服务 (localhost:3000)。

一、静态分析工具 — ✅ 全部适用于前端
------------------------------------

| 工具 | 前端价值 |
|---|---|
| dependency-cruiser | 分析 Vue 组件 / SDK 模块依赖关系，生成 SVG 依赖图 |
| Madge | 快速循环依赖检测 |
| TypeScript Compiler API / ts-morph | 分析组件间调用关系 |
| Knip | 检测未使用的导出、文件、依赖，对前端特别有用 |

二、运行时分析工具 — ❌ 不适用（Node.js 后端专用）
------------------------------------------------

应替换为前端专用工具：
  - Chrome DevTools Performance tab — 录制运行时性能
  - Lighthouse — 加载性能评分
  - Vue DevTools — 组件渲染性能、re-render 追踪

三、调用图 + 容斥计数 — ⚠️ 前端价值有限
----------------------------------------

前端瓶颈通常不在函数调用路径上，因为：
  - 组件树是事件驱动的，不像后端有固定请求 pipeline
  - 前端性能大头在：不必要的 re-render、包体积、API 请求瀑布

四、CPU 热点预估（原 3.3 节）— ❌ 完全不适用
---------------------------------------------

原文档列出的 app.ts（authz 中间件）、permission/service.ts、
event-bus/loop.ts、file-kv.ts 在该前端仓库中不存在，属于后端代码。
